# Module 1 — Install and Secure (8 deterministic checks)

Run each check in order. Emit one entry in the response `checks` array per check, in this exact order. Never skip a check. If a check cannot be performed for a transient reason, emit `pass: false` with the error in `detail`.

> **CLI assumptions:** the commands below assume the OpenClaw CLI surface documented as of v1.x. Concrete commands are flagged with `[CLI]`. If a command does not exist or returns unexpected output on the system, return `pass: false` with the error in `detail` rather than guessing. See `docs/cli-assumptions.md` for the list of commands this recipe relies on; verify against the live CLI before tagging.

---

## 1. `web-chat-responds`

Confirm the gateway is responding to local requests.

`[CLI]`
```
curl -sS -o /dev/null -w "%{http_code}" http://127.0.0.1:18789/v1/models -H "Authorization: Bearer $OPENCLAW_TOKEN"
```

(If `OPENCLAW_TOKEN` is not in env, read it from `~/.openclaw/credentials` via the gateway's documented mechanism — never display the token.)

- **pass:** HTTP 200
- **fail:** any other status, or curl error
- **detail (pass):** `"Gateway responding (HTTP 200) at 127.0.0.1:18789"`
- **detail (fail):** `"Gateway not responding: <status or error>"`
- **evidence:** `{ "status_code": <int>, "endpoint": "/v1/models" }`
- **fix_prompt:** `"Restart the gateway via the Hostinger dashboard, or run \`openclaw gateway restart\`."`

## 2. `claw-has-name`

Confirm the user's main workspace has a non-placeholder name in SOUL.md.

Read `~/.openclaw/workspaces/main/SOUL.md` (or the documented main-workspace path). Look for an Identity section and a `Name:` field (or equivalent). Reject any of these as not-set: empty string, `[YOUR_NAME]`, `TBD`, `null`, the literal string `your name here`.

- **pass:** name is set to a non-placeholder value
- **fail:** SOUL.md missing, no Identity/Name field found, or value is a placeholder
- **detail (pass):** `"Claw is named '<name>'"`
- **detail (fail):** one of `"SOUL.md missing"`, `"Identity section missing"`, `"Name unset or placeholder"`
- **evidence:** `{ "soul_path": "<path>", "name_found": "<value or null>" }`
- **fix_prompt:** `"Open SOUL.md and set a real name in the Identity section. Save when done."`

## 3. `audit-no-critical`

Run the built-in security audit and count critical failures.

`[CLI]`
```
openclaw security audit --json
```

(If `--json` flag is unsupported on this CLI version, fall back to plain-text output and grep for `CRITICAL`/`FAIL` markers, excluding documented expected-fails: firewall-in-container, control-UI port in 60000+ range.)

- **pass:** zero critical failures
- **fail:** at least one critical failure
- **detail (pass):** `"0 critical, <N> expected, <M> passed"`
- **detail (fail):** `"<K> critical failure(s): <comma-separated names>"`
- **evidence:** `{ "critical": [...], "expected_fail": [...], "passed": [...] }`
- **fix_prompt:** `"Restart the gateway and re-run the audit. Some fixes require a restart to apply."`

## 4. `gateway-bound`

Confirm gateway is bound to localhost and token auth is enabled.

`[CLI]`
```
openclaw config get gateway.bind
openclaw config get gateway.auth.mode
```

- **pass:** `gateway.bind == "127.0.0.1"` AND `gateway.auth.mode` is `"token"` or `"trusted-proxy"` (not `"open"` or unset)
- **fail:** anything else
- **detail (pass):** `"Bound to 127.0.0.1 with auth mode <mode>"`
- **detail (fail):** `"Bound to <addr> with auth mode <mode>"`
- **evidence:** `{ "bind": "<addr>", "auth_mode": "<mode>" }`
- **fix_prompt:** `"Set gateway.bind to 127.0.0.1 and ensure token auth is enabled, then restart the gateway."`

## 5. `dm-policy`

Confirm DM and group message policies are restrictive.

`[CLI]`
```
openclaw config get policy.dm
openclaw config get policy.group
```

- **pass:** both values are `"restricted"` (or any value stricter than `"open"`)
- **fail:** either is `"open"`, unset, or unknown
- **detail (pass):** `"DM policy: <val>; group policy: <val>"`
- **detail (fail):** `"DM policy: <val>; group policy: <val> — at least one is permissive"`
- **evidence:** `{ "dm": "<val>", "group": "<val>" }`
- **fix_prompt:** `"Set policy.dm and policy.group to restricted, then restart the gateway."`

## 6. `credentials-permissions`  *(platform-conditional)*

Verify `~/.openclaw/credentials` is owner-only readable.

Branch on `uname -s` (probed at start of run):

`[CLI]` Linux (`uname -s` = `Linux`):
```
stat -c "%a" ~/.openclaw/credentials
```

`[CLI]` macOS (`uname -s` = `Darwin`):
```
stat -f "%A" ~/.openclaw/credentials
```

- **pass:** output equals `700`
- **fail:** anything else, or file missing
- **detail (pass):** `"Permissions: 700 (owner-only)"`
- **detail (fail):** `"Permissions: <octal>; expected 700"` or `"~/.openclaw/credentials not found"`
- **evidence:** `{ "path": "~/.openclaw/credentials", "mode": "<octal>", "platform": "linux|macos" }`
- **fix_prompt:** `"Run \`chmod 700 ~/.openclaw/credentials\`."`

> **NEVER read or display the file contents.** This check inspects permissions only.

## 7. `web-search-disabled`

Confirm web_search is disabled in tool policy.

`[CLI]`
```
openclaw config get tools.web_search.enabled
```

- **pass:** value is `false`
- **fail:** value is `true`, unset, or unknown
- **detail (pass):** `"web_search disabled"`
- **detail (fail):** `"web_search is enabled (or policy unset)"`
- **evidence:** `{ "tools.web_search.enabled": <value> }`
- **fix_prompt:** `"Disable web_search in the tool policy. Module 7 will re-enable it deliberately later in the course."`

## 8. `heartbeat-zero`

Confirm gateway heartbeat is disabled.

`[CLI]`
```
openclaw config get gateway.heartbeat
```

- **pass:** value is `0`, `"0"`, `"0m"`, or otherwise indicates heartbeat off
- **fail:** any non-zero value, unset, or unknown
- **detail (pass):** `"Heartbeat disabled (0)"`
- **detail (fail):** `"Heartbeat: <value>"`
- **evidence:** `{ "gateway.heartbeat": <value> }`
- **fix_prompt:** `"Set gateway.heartbeat to 0 and restart the gateway."`
