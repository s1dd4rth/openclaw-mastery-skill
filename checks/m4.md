# Module 4 — Make It Proactive (3 deterministic)

Run each check in order. All three checks share data from one cron-list call — fetch once, evaluate three times.

> CLI assumption: `openclaw cron list --json` exists. The job is identified by name pattern matching `daily.reflection|reflection`. Verify before tagging.
>
> macOS reliability note: this validator confirms the cron job EXISTS and is correctly configured. It does NOT confirm the job actually fires on schedule (Mac mini sleep can suppress cron firing). That gap is a known limitation, see design doc Still Open #4.

Fetch the cron list once at the start:

```
CRONS=$(openclaw cron list --json)
JOB=$(echo "$CRONS" | jq '.[] | select(.name | test("daily.?reflection"; "i"))' | head -1)
```

If `$JOB` is empty, all three checks fail.

## 1. `cron-exists`

- **pass:** `$JOB` is non-empty
- **fail:** no cron job matches the daily-reflection pattern
- **detail (pass):** `"Daily reflection cron job found (name: <name>, id: <id>)"`
- **detail (fail):** `"No cron job matching 'daily reflection' pattern"`
- **evidence:** `{ "matched_job": { "name": "<name>", "id": "<id>" } }` or `null`
- **fix_prompt:** `"Tell the Claw: 'Create the daily reflection cron job now. Run it once manually so I can verify delivery.'"`

## 2. `cron-schedule`

- **pass:** `$JOB` exists AND has both a non-empty `schedule` (cron expression) AND a non-empty `timezone` field
- **fail:** missing job, missing schedule, or missing timezone
- **detail (pass):** `"Schedule: <cron-expression>; timezone: <tz>"`
- **detail (fail):** `"Schedule or timezone missing or invalid"`
- **evidence:** `{ "schedule": "<expr>", "timezone": "<tz>" }`
- **fix_prompt:** `"Tell the Claw: 'Update the daily reflection cron expression and timezone to match my chosen time and timezone.'"`

## 3. `cron-telegram`

- **pass:** `$JOB.delivery.channel == "telegram"`
- **fail:** delivery.channel is anything else, or unset
- **detail (pass):** `"Delivery channel: telegram"`
- **detail (fail):** `"Delivery channel: <channel>"` or `"delivery.channel unset"`
- **evidence:** `{ "delivery_channel": "<channel>" }`
- **fix_prompt:** `"Tell the Claw: 'Update the daily reflection cron job to deliver via Telegram.'"`
