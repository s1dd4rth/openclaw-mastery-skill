# Module 9 — Give It a Team (5 deterministic + 2 judgment)

Run each check in order. Always include manual checks with `pass: null, manual: true`.

> CLI assumptions: `openclaw agents list --json` returns named agents with workspace paths. Writer workspace path is `~/.openclaw/workspaces/writer/`. `openclaw config get gateway.agent_comms` (or equivalent) controls agent-to-agent comms. Verify before tagging.

## 1. `writer-exists`

Confirm a named agent called `writer` exists on this gateway.

```
openclaw agents list --json | jq '.[] | select(.name == "writer")'
```

- **pass:** entry present
- **fail:** no agent named `writer`
- **detail (pass):** `"writer agent present (model: <model>, workspace: <path>)"`
- **detail (fail):** `"No agent named 'writer'"`
- **evidence:** `{ "agent_present": <bool>, "model": "<model>", "workspace_path": "<path>" }`
- **fix_prompt:** `"Tell the Claw: 'Create the writer agent workspace now with a capable model and full identity files.'"`

## 2. `writer-soul-exists`

Confirm the writer workspace contains a SOUL.md (existence only — voice quality is check #6, manual).

```
test -f ~/.openclaw/workspaces/writer/SOUL.md
```

- **pass:** file exists
- **fail:** missing
- **detail (pass):** `"writer/SOUL.md present"`
- **detail (fail):** `"writer/SOUL.md missing"`
- **evidence:** `{ "soul_path": "<path>", "exists": <bool>, "size_bytes": <int> }`
- **fix_prompt:** `"Tell the Claw: 'Create SOUL.md in the writer workspace with at least an Identity and Voice section.'"`

## 3. `writer-identity-files`

Confirm all four identity files are present in the writer workspace.

```
for f in SOUL.md USER.md MEMORY.md AGENTS.md; do
  test -f ~/.openclaw/workspaces/writer/$f && echo "ok $f" || echo "MISSING $f"
done
```

- **pass:** all four files present
- **fail:** any missing
- **detail (pass):** `"All four identity files present (SOUL, USER, MEMORY, AGENTS)"`
- **detail (fail):** `"Missing: <comma-separated>"`
- **evidence:** `{ "files_present": [<list>], "files_missing": [<list>] }`
- **fix_prompt:** `"Tell the Claw: 'Create USER.md, AGENTS.md, and MEMORY.md in the writer workspace with appropriate defaults.'"`

## 4. `agent-comms-enabled`

Confirm agent-to-agent communication is enabled between main and writer.

```
openclaw config get gateway.agent_comms.enabled
openclaw config get gateway.agent_comms.peers
```

- **pass:** `enabled` is `true` AND `peers` includes both `main` and `writer` (or equivalent permissive config)
- **fail:** disabled, or peers list does not include both
- **detail (pass):** `"Agent comms enabled, main↔writer peering configured"`
- **detail (fail):** specific reason
- **evidence:** `{ "enabled": <bool>, "peers": [<list>] }`
- **fix_prompt:** `"Tell the Claw: 'Enable agent-to-agent communication with the writer workspace.'"`

## 5. `delegation-rule`

Confirm the main workspace AGENTS.md has a writer delegation rule.

```
grep -iE '(use|delegate to) (the )?writer (agent|workspace).*(long.?form|writing|draft)' ~/.openclaw/workspaces/main/AGENTS.md
```

- **pass:** at least one matching line
- **fail:** none
- **detail (pass):** `"AGENTS.md has writer delegation rule"`
- **detail (fail):** `"AGENTS.md missing writer delegation rule"`
- **evidence:** `{ "matched_lines": <count> }`
- **fix_prompt:** `"Tell the Claw: 'Add a delegation rule to main AGENTS.md: use the writer agent for any long-form content over 300 words.'"`
