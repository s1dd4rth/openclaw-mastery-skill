# Module 2 — Make It Personal (4 deterministic + 1 attestation)

Run each check in order. Emit one entry per check, in this exact order. Follow the JSON contract in [`SKILL.md`](../SKILL.md). Always include the manual checks with `pass: null, manual: true`.

> CLI assumptions: this recipe assumes a stable workspace path at `~/.openclaw/workspaces/main/`. Verify against a live install before tagging — see [`docs/cli-assumptions.md`](../docs/cli-assumptions.md).

## 1. `soul-exists`

Check that SOUL.md exists in the main workspace and contains a `Hard Limits` section header.

```
test -f ~/.openclaw/workspaces/main/SOUL.md && grep -E '^## *Hard Limits' ~/.openclaw/workspaces/main/SOUL.md
```

- **pass:** file exists AND a `## Hard Limits` (or `### Hard Limits`) heading is present
- **fail:** file missing, or no Hard Limits section
- **detail (pass):** `"SOUL.md present with Hard Limits section"`
- **detail (fail):** one of `"SOUL.md missing"`, `"SOUL.md present but Hard Limits section missing"`
- **evidence:** `{ "path": "<path>", "has_hard_limits": <bool>, "size_bytes": <int> }`
- **fix_prompt:** `"Open SOUL.md and add a '## Hard Limits' section with absolute rules. Save when done."`

## 2. `user-exists`

Check that USER.md exists in the main workspace and contains a non-empty `Name:` field and a non-empty `Focus:` (or `Focus area:`) field.

```
test -f ~/.openclaw/workspaces/main/USER.md
grep -iE '^name:' ~/.openclaw/workspaces/main/USER.md
grep -iE '^focus( area)?:' ~/.openclaw/workspaces/main/USER.md
```

- **pass:** file exists AND both fields present with non-empty, non-placeholder values (reject `[YOUR_NAME]`, `TBD`, `your name`, `your focus`)
- **fail:** file missing, or either field missing/placeholder
- **detail (pass):** `"USER.md present with name and focus area"`
- **detail (fail):** specific reason
- **evidence:** `{ "path": "<path>", "name_present": <bool>, "focus_present": <bool> }`
- **fix_prompt:** `"Open USER.md and set a real Name: and Focus: line. Keep the file under 500 words."`

## 3. `memory-exists`

Check that MEMORY.md exists and has the three structural section headers: Decisions, Preferences, Open Loops.

```
test -f ~/.openclaw/workspaces/main/MEMORY.md
grep -cE '^## *(Decisions|Preferences|Open Loops)' ~/.openclaw/workspaces/main/MEMORY.md
```

- **pass:** file exists AND match count == 3 (all three headers present)
- **fail:** file missing or fewer than 3 of the required headers
- **detail (pass):** `"MEMORY.md initialized with Decisions, Preferences, Open Loops sections"`
- **detail (fail):** `"MEMORY.md missing or sections incomplete (<N>/3 found)"`
- **evidence:** `{ "path": "<path>", "headers_present": <count>, "missing_headers": [<list>] }`
- **fix_prompt:** `"Create MEMORY.md with Decisions, Preferences, and Open Loops sections, each with a placeholder entry."`

## 4. `agents-exists`

Check that AGENTS.md exists and contains a `Startup Checklist` section.

```
test -f ~/.openclaw/workspaces/main/AGENTS.md
grep -E '^## *Startup Checklist' ~/.openclaw/workspaces/main/AGENTS.md
```

- **pass:** file exists AND `## Startup Checklist` heading present
- **fail:** file missing or section missing
- **detail (pass):** `"AGENTS.md present with Startup Checklist section"`
- **detail (fail):** specific reason
- **evidence:** `{ "path": "<path>", "has_startup_checklist": <bool> }`
- **fix_prompt:** `"Open AGENTS.md and add a '## Startup Checklist' section listing the files to load at session start."`

## 5. `identity-durable` *(manual — attestation)*

This check verifies the user typed `/new` and confirmed the identity loaded correctly in a fresh session. The validator cannot observe whether the user actually started a fresh session.

Always emit:

```json
{
  "id": "identity-durable",
  "pass": null,
  "detail": "Manual: user confirms identity loaded correctly in a fresh /new session",
  "evidence": null,
  "fix_prompt": null,
  "manual": true
}
```
