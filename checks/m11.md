# Module 11 (Bonus) — Anchor Workspace in Obsidian Vault (3 deterministic + 1 attestation)

Run each check in order. Emit one entry per check, in this exact order. Always include the manual check with `pass: null, manual: true`. All checks are filesystem-only — no OpenClaw CLI dependencies.

> Prerequisite: Module 2 must be done (the identity files we verify must exist in the workspace).
>
> Bonus modules are NOT included in M10's completion code orchestration. `runModule10` iterates `verify_module(1..9)` only; the M10 completion code is a function of M1–M9 results, deliberately unaffected by bonus modules so existing learners' codes remain stable.

## 1. `workspace-is-symlink`

`~/.openclaw/workspace` exists AND is a symbolic link (not a real directory).

```
node -e 'console.log(require("node:fs").lstatSync(require("node:os").homedir()+"/.openclaw/workspace").isSymbolicLink())'
```

- **pass:** path exists AND lstat reports symbolic link
- **fail:** path missing OR path is a real directory
- **detail (pass):** `"~/.openclaw/workspace is a symbolic link"`
- **detail (fail):** `"~/.openclaw/workspace not found — run the do-prompt to anchor it in your vault."` or `"~/.openclaw/workspace is a real directory, not a symlink — workspace is still in its original location."`
- **evidence:** `{ "path": "~/.openclaw/workspace", "exists": <bool>, "is_symlink": <bool> }`
- **fix_prompt:** `"Tell the Claw: 'Anchor my OpenClaw workspace inside my Obsidian vault.'"`

## 2. `workspace-target-in-vault`

The symlink target's resolved real path is inside an Obsidian vault. A vault is detected by the presence of an `.obsidian/` directory at the resolved path or any of its ancestors (the canonical Obsidian marker).

```
realpath ~/.openclaw/workspace
# then walk upward from that path looking for `.obsidian/`
```

- **pass:** `.obsidian/` directory found at some ancestor of the resolved target
- **fail:** symlink broken OR no `.obsidian/` found in any ancestor
- **detail (pass):** `"Workspace target is inside Obsidian vault at <vault-root>"`
- **detail (fail):** `"Workspace symlink points at <resolved>, but no Obsidian vault was found in any ancestor (looked for .obsidian/)."`
- **evidence:** `{ "workspace_path": "~/.openclaw/workspace", "resolved_target": "<path>", "vault_root": "<path or null>" }`
- **fix_prompt:** `"Make sure the symlink target is inside a real Obsidian vault (vault root contains an .obsidian/ directory)."`

## 3. `identity-files-readable`

After the move, all four identity files are readable at `~/.openclaw/workspace/<name>.md` (i.e., OpenClaw still finds them through the symlink). Reuses M2's identity-file existence pattern.

```
test -f ~/.openclaw/workspace/SOUL.md
test -f ~/.openclaw/workspace/USER.md
test -f ~/.openclaw/workspace/MEMORY.md
test -f ~/.openclaw/workspace/AGENTS.md
```

- **pass:** all four present
- **fail:** any missing
- **detail (pass):** `"All four identity files readable through the symlink (SOUL, USER, MEMORY, AGENTS)"`
- **detail (fail):** `"Missing through the symlink: <comma-separated>"`
- **evidence:** `{ "files_present": [...], "files_missing": [...] }`
- **fix_prompt:** `"If the files exist in the vault but aren't reachable here, the symlink target may be wrong. Verify with \`readlink ~/.openclaw/workspace\` and \`ls -la $(readlink ~/.openclaw/workspace)\`."`

## 4. `mobile-access-works` *(manual — attestation)*

Requires the user to open Obsidian on their phone (with the vault synced via Obsidian Sync / iCloud / Syncthing — their choice) and confirm SOUL.md is readable. The validator can't observe a phone.
