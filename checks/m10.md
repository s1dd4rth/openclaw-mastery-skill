# Module 10 — What Comes Next (3 deterministic + 1 attestation)

This module is **meta**: the validator orchestrates self-review by calling `verify_module(N)` for each prior module and aggregating results.

Run check 1 first to populate the orchestration data; checks 2 and 3 derive from it. Always include the manual check with `pass: null, manual: true`.

## 1. `claw-reviewed-setup`

Run `verify_module(N)` for each N in 1..9 and collect the results.

```
for n in 1 2 3 4 5 6 7 8 9; do
  RESULT_n=$(invoke openclaw-mastery.verify_module module=$n)
done
```

Aggregate per module: count `pass: true`, `pass: false`, and `pass: null` (manual).

- **pass:** all 9 prior-module verifications completed without error (regardless of individual check pass/fail — this check verifies the REVIEW happened, not that everything is green)
- **fail:** one or more module verifications could not be invoked or produced invalid output
- **detail (pass):** `"Reviewed setup across all 9 prior modules"`
- **detail (fail):** `"Module(s) <N,M> verification failed to run: <reason>"`
- **evidence:** `{ "modules_reviewed": <int>, "errors": [{ "module": N, "reason": "..." }] }`
- **fix_prompt:** `"Tell the Claw: 'Re-run openclaw-mastery validator for each module 1–9 and report which produced invalid output.'"`

## 2. `completion-report`

Generate a per-module pass/fail summary from the data collected in check 1.

For each module 1..9, compute: `passed = sum(checks where pass==true)`, `failed = sum(checks where pass==false)`, `manual = sum(checks where pass==null)`, `total = passed + failed + manual`.

- **pass:** report data is fully formed for all 9 modules (no missing rows)
- **fail:** report data missing or partial
- **detail (pass):** `"Per-module completion report generated (M1..M9)"`
- **detail (fail):** `"Report incomplete: missing data for module(s) <N>"`
- **evidence:**
  ```json
  {
    "report": [
      { "module": 1, "passed": 8, "failed": 0, "manual": 0, "total": 8 },
      { "module": 2, "passed": 4, "failed": 0, "manual": 1, "total": 5 },
      "..."
    ]
  }
  ```
- **fix_prompt:** `"Tell the Claw: 'Re-run the course review and produce a per-module pass/fail report.'"`

## 3. `completion-code`

Produce a deterministic, short completion code from the report data — a hash of the per-module pass tuple.

Suggested format: `OCM-<base32 of sha256(json_canonical([{m, passed, failed, manual} for m in 1..9]))>` truncated to 12 chars.

- **pass:** code generated successfully and matches a recompute against the report data
- **fail:** code missing or mismatched
- **detail (pass):** `"Completion code: <code>"`
- **detail (fail):** `"Could not generate completion code"`
- **evidence:** `{ "code": "<code>", "based_on_module_count": 9 }`
- **fix_prompt:** `"Tell the Claw: 'Generate the completion code based on the course review results.'"`

## 4. `assessment-opened` *(manual — attestation)*

Whether the user opened the Google Form assessment cannot be observed by the validator.

Always emit:

```json
{
  "id": "assessment-opened",
  "pass": null,
  "detail": "Manual: user confirms they opened the Google Form assessment",
  "evidence": null,
  "fix_prompt": null,
  "manual": true
}
```
