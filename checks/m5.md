# Module 5 — Give It Skills (3 deterministic + 1 attestation)

Run each check in order. Always include the manual check with `pass: null, manual: true`.

> CLI assumption: `openclaw skills list --json` exists and lists installed workspace skills. The workspace skills directory path is `~/.openclaw/workspaces/main/skills/<skill-name>/SKILL.md`. Verify before tagging.

## 1. `doc-summary-installed`

Confirm `document-summary` skill is installed for the main workspace.

```
openclaw skills list --json --workspace main | jq '.[] | select(.name == "document-summary")'
```

- **pass:** at least one entry returned
- **fail:** empty result
- **detail (pass):** `"document-summary installed (version: <ver>)"`
- **detail (fail):** `"document-summary not installed"`
- **evidence:** `{ "installed": <bool>, "version": "<ver>" }`
- **fix_prompt:** `"Tell the Claw: 'Install document-summary for this workspace now.'"`

## 2. `quick-note-exists`

Confirm `quick-note` is a workspace skill (its own SKILL.md present).

```
test -f ~/.openclaw/workspaces/main/skills/quick-note/SKILL.md
```

- **pass:** file exists
- **fail:** file missing
- **detail (pass):** `"quick-note workspace skill present"`
- **detail (fail):** `"quick-note workspace skill missing"`
- **evidence:** `{ "skill_path": "<path>", "exists": <bool> }`
- **fix_prompt:** `"Tell the Claw: 'Create the quick-note SKILL.md in the workspace skills folder now.'"`

## 3. `both-skills-work`

Confirm both `document-summary` and `quick-note` are listed as active in the current session.

```
openclaw skills list --json --workspace main --active
```

- **pass:** both `document-summary` AND `quick-note` are active
- **fail:** either is missing or inactive
- **detail (pass):** `"Both document-summary and quick-note active"`
- **detail (fail):** `"<which one(s)> not active"`
- **evidence:** `{ "doc_summary_active": <bool>, "quick_note_active": <bool> }`
- **fix_prompt:** `"In OpenClaw, type /new to start a fresh session and confirm both skills load."`

## 4. `skills-fresh-session` *(manual — attestation)*

The validator cannot detect whether the user typed `/new` before testing.

Always emit:

```json
{
  "id": "skills-fresh-session",
  "pass": null,
  "detail": "Manual: user confirms they typed /new before testing the new skills",
  "evidence": null,
  "fix_prompt": null,
  "manual": true
}
```
