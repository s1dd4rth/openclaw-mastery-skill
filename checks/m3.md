# Module 3 — Connect a Channel (1 deterministic + 1 attestation)

Run each check in order. Emit one entry per check. Always include the manual check with `pass: null, manual: true`.

> CLI assumption: `openclaw channels list` exists and emits structured output. Verify before tagging.

## 1. `telegram-connected`

List configured channels and verify a Telegram channel is present and active.

```
openclaw channels list --json
```

(If `--json` is unsupported, fall back to plain text and grep for `telegram` AND `active|connected|ready`.)

- **pass:** at least one channel of type `telegram` is in an active/connected state
- **fail:** no telegram channel configured, or only in pending/error state
- **detail (pass):** `"Telegram connected (channel id: <id>, status: <status>)"`
- **detail (fail):** `"Telegram not connected"` or specific status
- **evidence:** `{ "channels": [{"type": "<t>", "status": "<s>"}, ...] }`
- **fix_prompt:** `"In OpenClaw, generate a fresh Telegram pairing code and re-pair from your phone. Pairing codes expire quickly — have the app ready before you start."`

## 2. `telegram-responds` *(manual — attestation)*

Requires the user to send a message from their phone via Telegram. The validator cannot inject messages from outside.

Always emit:

```json
{
  "id": "telegram-responds",
  "pass": null,
  "detail": "Manual: user sends a message from phone via Telegram and confirms the Claw replies",
  "evidence": null,
  "fix_prompt": null,
  "manual": true
}
```
